# SaaS Homepage – Design Analysis

## Navbar and Hero Layout

- The homepage starts with a clean and minimal navigation bar.
- The logo is positioned on the left side.
- Navigation links are placed on the right along with a primary call-to-action button.
- The hero section follows a two-column layout on desktop.
- The left column contains the main heading, description, and action buttons.
- The right column displays a dashboard-style analytics preview.
- On mobile devices, the layout changes to a single-column structure with stacked content.

---

## Typography and Colour Palette

### Typography
- Large bold heading for the hero section.
- Medium-sized paragraph text for descriptions.
- Clear font hierarchy throughout the page.
- Consistent font weights for headings, subheadings, and body text.

### Colour Palette
- Primary: Light Blue Gradient
- Secondary: White
- Text Primary: Black
- Text Secondary: Gray
- Accent: Yellow
- Buttons: Dark Gray / Black


## Buttons and Visual Hierarchy

### Primary Button
- Dark background
- White text
- Rounded corners
- Hover effect

### Secondary Button
- Transparent or outlined style
- Dark border
- Hover effect

### Visual Hierarchy
1. Hero Heading
2. Dashboard Preview
3. Call-to-Action Buttons
4. Section Headings
5. Cards
6. Supporting Text


## Container Width and Section Spacing

- Maximum container width keeps the layout centered.
- Consistent horizontal padding on all screen sizes.
- Large vertical spacing between sections.
- Responsive margins and gaps improve readability.
- No horizontal scrolling.

## Desktop and Mobile Behaviour

### Desktop (1280px+)
- Two-column hero layout
- Full navigation menu
- Large images
- Multi-column cards

### Tablet (800px–1279px)
- Reduced spacing
- Scaled images
- Two-column sections where appropriate

### Mobile (1px–799px)
- Single-column layout
- Hamburger navigation
- Full-width buttons
- Responsive images
- Cards stacked vertically
- Optimized spacing for smaller screens


## Accessibility

- Semantic HTML elements
- Keyboard-focus states on buttons and links
- Proper heading hierarchy
- Responsive design for different devices
- Accessible color contrast
